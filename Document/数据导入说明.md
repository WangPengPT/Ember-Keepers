# 数据导入说明

## 概述

本项目使用CSV格式存储游戏数据，并通过Unity Editor工具一键导入到游戏中。

## 数据文件位置

所有CSV数据文件位于 `Document/` 目录下：
- `Heroes.csv` - 英雄数据（12个英雄）
- `Equipment.csv` - 装备数据（50件装备）
- `Monsters.csv` - 怪物数据（30个普通怪物 + 6个精英怪 + 3个Boss）
- `Waves.csv` - 关卡数据（20波主线 + 10波无尽模式）

## 导入步骤

1. 打开Unity编辑器
2. 在菜单栏选择 `EmberKeepers` -> `数据导入工具`
3. 点击 `一键导入所有数据` 按钮
4. 等待导入完成，数据将自动保存到 `Assets/Resources/Data/` 目录下

## 数据使用

导入后，数据会自动被 `DataManager` 加载。在代码中使用方式：

```csharp
// 获取英雄数据
HeroData hero = DataManager.Instance.GetHeroData("hero_fire_dps_ignis");

// 获取装备数据
EquipmentData equipment = DataManager.Instance.GetEquipmentData("weapon_starfire_core");

// 获取怪物数据
MonsterData monster = DataManager.Instance.GetMonsterData("monster_whisper_lvl1");

// 获取关卡数据
WaveDataEntry wave = DataManager.Instance.GetWaveData(1);

// 查询方法
List<HeroData> fireHeroes = DataManager.Instance.GetHeroesByElement(ElementType.Fire);
List<EquipmentData> weapons = DataManager.Instance.GetEquipmentByType(EquipmentType.Weapon);
List<MonsterData> elites = DataManager.Instance.GetEliteMonsters();
```

## 数据格式说明

### Heroes.csv
- `heroId`: 英雄唯一ID
- `elementType`: 元素类型 (Fire/Ice/Thunder/Earth)
- `heroClass`: 职业类型 (Guardian/DPS/Support)
- 其他字段为英雄的基础属性和技能信息

### Equipment.csv
- `equipmentId`: 装备唯一ID
- `rarity`: 稀有度 (Common/Uncommon/Rare/Epic/Legendary)
- `equipmentType`: 装备类型 (Weapon/Armor/Trinket)
- `requiredClass`: 需求职业 (None表示全职业)
- `effectIds`: 词条效果ID列表（逗号分隔）

### Monsters.csv
- `monsterId`: 怪物唯一ID
- `family`: 怪物家族 (Whisper/Corroder/Bonewall/Aberration/Cluster/ChaosMage/None)
- `isElite`: 是否为精英怪
- `isBoss`: 是否为Boss
- 其他字段为怪物的基础属性和抗性

### Waves.csv
- `waveNumber`: 波次编号
- `difficulty`: 难度系数
- `hasBoss`: 是否有Boss
- `bossId`: Boss ID（如果有）
- `monsterSpawns`: 怪物生成配置（格式：monsterId:count:level:spawnDelay:spawnInterval，多个用逗号分隔）

## 注意事项

1. 导入前请确保CSV文件格式正确
2. 导入会覆盖现有的ScriptableObject资源
3. 修改CSV后需要重新导入才能生效
4. 数据ID必须唯一，否则会覆盖
